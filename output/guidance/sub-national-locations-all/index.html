Geography - sub-national locations (all elements)

This table lists all elements that can be included in the location element. It includes all the recommended elements, as outlined on the geography - sub-national locations page, plus further elements that can be used to build a comprehensive picture of an activity’s location, if known.

Technical guidance summary

Element

Use

Rules

Guidance
location Details the geographical locations in which the activity is taking place. The @ref attribute should be included; this is an internal reference for each location element. This can be used to link one of the activity’s results to a defined location.
location-reach Specifies whether the location is where the activity is happening or where the intended beneficiaries live. This can only be published once. The value must be on the geographic location reach codelist.
location-id A unique code describing the location according to a recognised gazetteer or repository of administrative boundaries. The must come from one of the lists on the geographic vocabulary codelist. Administrative areas should be published here only if the location being defined is the administrative area itself. For describing the administrative area/s within which a more specific location falls, the location/administrative element should be used.
name The name of the location This can only be published once. The name can be repeated in multiple languages.
description A description of the location This can only be published once. The description can be repeated in multiple languages.
activity-description A description that qualifies the activity taking place at the location. This can only be published once. This should be used when the activity has multiple locations; can be used to specify what activity is happening at each location. This should not duplicate information provided elsewhere.
administrative A unique code describing the location according to an administrative boundary repository. This must come from one of the lists on the geographic vocabulary codelist.
point and pos This holds the geo-coordinates for the location, in the format of latitude and longitude. The point element must contain the @srsName attribute with the value: http://www.opengis.net/def/crs/EPSG/0/4326” The latitude and longitude coordinates are published within the pos element.
exactness Specifies whether the location provided is exact or approximate. This value must be on the geographic exactness codelist.
location-class Specifies the form of the location e.g. populated place (town, farm), or topographical feature (mountain, river). This value must be on the geographic location class codelist.
feature-designation This adds more detail to the type of location (e.g. beach, well or college). This value must be on the location type codelist.
